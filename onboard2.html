Step 2: Set Up Google Sign-In & Domain Configuration

1 Go to Authentication

In your Firebase Console, navigate to the Authentication section:

  • Click Build ‚Üí Authentication in the left sidebar
  • Click the "Get started" button if you haven't done so already

2 Enable Google Sign-In

Configure Google as a sign-in provider:

  • Click the Sign-in method tab at the top
  • Find Google in the list of providers
  • Click on Google ‚Üí a configuration page will open

3 Configure Google Sign-In

Fill in the required configuration fields:

  • Project support email: Pick your Google Account email ‚Äî Firebase uses this for OAuth consent
  • Public-facing name: By default, it's your Firebase project's name. You can edit it if you want
  • Authorized domains: This part is very important ‚Äî Firebase needs to know which website domains are allowed to use Google Sign-In

4 Add Authorized Domains

Under Authorized domains, you'll see the default domains:

Project domains: - your-project-id.firebaseapp.com - localhost

These are there by default and allow you to test locally.

‚úÖ For Local Development:

You can test Google Sign-In on localhost right away ‚Äî no extra domain is needed.

‚úÖ For Custom Domain Deployment:

If you're deploying your site on a custom domain (e.g., www.myawesomeapp.com):

  • Click Add domain
  • Enter www.myawesomeapp.com
  • Click Add

üîπ Important Tips:

  • Only add the domain ‚Äî no https:// or / path
  • If you use subdomains, add each one if needed (like app.myawesomeapp.com)

5 Save Changes

When done configuring all settings:

  • Scroll down to the bottom of the page
  • Click Save to enable Google as a sign-in provider

6 How it Works

When your frontend calls Firebase's Google Auth Provider, the sign-in popup will only work on:

  • localhost (for development)
  • your-project-id.firebaseapp.com (Firebase Hosting)
  • Any custom domains you added

‚ö†Ô∏è Important Note:

If you don't add your domain, the sign-in popup will fail with an "unauthorized domain" error.
